Module 4: Constrain developer card input length (Ticket 402)

Test steps:

1: Build and run CiCo.
2: Attempt to type fewer than the required number of digits into the
  developer card field (e.g., type 123)

3: Attempt to type more than the required number of
  digits into the developer card field (e.g., paste or type 123456789)

4: Attempt to type non‑digit characters (letters, symbols, spaces)

5: Enter exactly the required number of digits (e.g., 00000000)
  and click Update

Expected results
- In step 2, the field should allow typing only up
  to the required number of digits; no error should
  occur, but the field should not auto‑pad or accept
  incomplete input as valid.

- In step 3, the field should reject any additional
  digits beyond the required length; the extra characters
  should not appear in the field.

- In step 4, the field should reject all non‑digit
  characters; they should not appear in the field.

- In step 5, the input should be accepted normally
  and the system should proceed exactly as it does for
  a valid card number in other tests.


Module 5: Quit the Application when the window is Closed (Ticket 502)

Test Steps:

1. Build and run CiCo
2. Observe the terminal or shell window where CiCo was launched.
3. Close the CiCo window using the window’s close button (the red x in the title bar).
4. Wait 1–2 seconds.

Expected Results:

- After step 3, the CiCo application should fully exit
  instead of running in the background.
- The shell or terminal should display the message “Clean shutdown”,
  confirming that the shutdown hook was executed.
- No Java process should remain active after the window is closed.
